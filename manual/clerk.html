Clerk Agent

The Clerk is the first agent in the pipeline. It reads each party's legal document and extracts a complete, structured summary — facts, legal issues, citations, and prayers — ready for adversarial comparison.

Role and Responsibility

The Clerk's job is purely extraction — it does not interpret, compare, or argue. It reads one document at a time and produces a StandardizedPartySubmission: a consistent data structure that every downstream agent expects.

Both the Petitioner's and Respondent's documents are processed in parallel by separate Clerk calls — cutting the processing time roughly in half.

🗂️
Think of the Clerk as a highly trained court reader: it goes through the document page by page, notes every factual statement with its page reference, records every law and case cited, identifies every legal issue the party raises, and lists what relief they are asking for.

Output — StandardizedPartySubmission

The Clerk produces a structured JSON object saved to the database. Here is what each field means:

extracted_facts list of {statement, page_index, verified} Each material factual claim made by the party, anchored to the page it appears on. verified: true means the statement text was found verbatim or near-verbatim in the source. issues_raised list of strings Every distinct legal issue explicitly raised by this party, in their own framing. These feed the Registrar's issue-comparison step. cited_laws_and_cases list of {citation, page_index, verified} All statutes, sections, rules, and case citations mentioned. Exact citation string as written in the document, with source page. prayers list of strings The specific reliefs or remedies this party asks the court to grant. Extracted verbatim where possible.

Reading the Clerk Tab

The Clerk tab shows two columns side by side — Petitioner (blue header) and Respondent (red header). Each column has four collapsible sections:

Facts
Each fact shows a page tag (e.g. p.3) on the left. These page numbers correspond to the source PDF. Use the party toggle in the PDF viewer to jump to that page and verify the extraction.
Issues Raised
The legal questions this party says are at stake. Compare these between columns — overlapping issues become "framed issues" in the Registrar; non-overlapping ones become single-party issues.
Citations
Every statute and case cited. A citation without a page tag means it was mentioned but the Clerk could not pin it to a page. Check manually if the citation is important.
Prayers
The relief sought by each party. These are important — they define the scope of what the Judge can award and should reflect the actual pleadings exactly.

Quality Checklist

Before proceeding to the Registrar, verify the Clerk output against these criteria:

✓
Core facts are captured
All key factual events — dates, names, amounts, statutory provisions involved — appear in the Facts list. If a critical fact is missing, the document text may be incomplete.
✓
Issues match the document's arguments
The issues listed should match what the party actually argues, not a generic restatement. If issues appear vague or invented, the document text did not explicitly raise them.
✓
Key citations are present
Every statute and important case cited in the filing should appear. Cross-check against the document in the PDF viewer.
✓
Prayers are specific
Each prayer should be a distinct, specific relief — not a generic "any other order". If prayers appear vague, check the filing's Prayer/Relief section.

Re-running the Clerk

If the output is unsatisfactory, you can re-run the Clerk by going back to the Setup tab and clicking ▶ Run Clerk Agent again. This overwrites the previous output.

Common reasons to re-run:

  • You attached the wrong document — detach and re-attach the correct one first.
  • The document type was wrong (e.g. set as "Petition" when it is a "Rejoinder") — change the type and re-run.
  • PageIndex was not previously run on the PDF — run PageIndex, then re-run the Clerk.
⚠️
Re-running Clerk resets the pipeline. If the Registrar has already run, re-running the Clerk will overwrite the clerk output but will not automatically re-run the Registrar. You must manually trigger the Registrar again after a Clerk re-run.

Next Step

Once both columns show complete output and you are satisfied with the quality, click ▶ Run Registrar Agent at the bottom of the Clerk tab. The pipeline will advance to the Registrar stage.
