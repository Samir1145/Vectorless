End-to-End Pipeline Walkthrough

A detailed, step-by-step guide to running a case through the full adversarial pipeline — from first upload to final exported order.

Before You Start

Ensure the following before creating a case:

ℹ️
Prerequisites:
  • Both party documents are uploaded as PDFs and have a done status in the Documents panel.
  • PageIndex has processed both PDFs (the tree index has been built — this extracts the text needed by the Clerk Agent).
  • The backend server is running (python3 server.py) and an OpenAI API key is configured in .env.

Step 1 — Create the Case

1
Open the Cases panel
In the left sidebar, scroll past the document list to find the Cases section. Click the + button in the section header.
2
Enter a case title
Type a descriptive title in the format "Petitioner Name v. Respondent Name" — e.g. Ramesh Kumar v. Municipal Corporation of Delhi. Press Enter or click Create.
3
The case opens in the right panel
The right panel switches to Case Mode and lands on the Setup tab. The PDF panel shows a Petitioner/Respondent toggle at the top.

Step 2 — Attach Documents (Setup Tab)

In the Setup tab you will see two slots: Petitioner and Respondent.

1
Attach the Petitioner's document
Click the dropdown under Petitioner and select the correct PDF from your uploaded documents. Only done documents appear.
2
Attach the Respondent's document
Repeat for the Respondent slot. The document type (Petition, Reply, Rejoinder, Affidavit, Exhibit, Other) is pre-set and can be changed.
3
Verify and run
Once both slots show a document, the ▶ Run Clerk Agent button activates. Click it.
⚠️
Important: The Clerk Agent reads the text extracted by PageIndex from the PDF (stored in the database). If a document shows done but PageIndex was never run on it, the Clerk will receive empty text. Always ensure both documents have been fully processed by PageIndex before running the pipeline.

Step 3 — Clerk Agent (Clerk Tab)

The Clerk runs both documents simultaneously in parallel. Status transitions:

pending → clerk_running → clerk_done

While the Clerk is running, the progress bar shows a spinning indicator. Switch to the Clerk tab to monitor progress. Once done:

  • Two columns appear — Petitioner on the left, Respondent on the right.
  • Each column shows Facts (with page numbers), Issues Raised, Citations, and Prayers.
  • Review the output. If something looks wrong, check whether PageIndex was run on the PDF.
  • When satisfied, click ▶ Run Registrar Agent at the bottom of the tab.
💡
Reviewing Clerk output: Page numbers on each fact are clickable anchors (when the PDF is loaded in the viewer). Click the Petitioner / Respondent toggle in the PDF panel header to switch between the two source documents while reviewing the Clerk tab.

Step 4 — Registrar Agent (Registrar Tab)

clerk_done → registrar_running → review_pending

The Registrar reads both clerk outputs and produces the Adversarial Matrix — the central document of the pipeline. Switch to the Registrar tab once it finishes.

What to review in the Registrar tab

Undisputed Background
Facts that neither party contests. These become the narrative background of the order. Check for factual accuracy — a disputed fact should not appear here.
Framed Issues
Every contested legal question, stated neutrally. Expand each issue to see both stances mapped side-by-side. Are all key issues captured? Is any issue biased toward one party?

The Human Review Gate

At the bottom of the Registrar tab you will find two buttons:

✓ Approve
The matrix accurately captures all issues with balanced stances. The Judge Agent may proceed. This action is irreversible without re-running the Registrar.
✗ Reject
The matrix is inaccurate or incomplete. The pipeline is blocked. You can re-run the Registrar after making corrections (e.g. re-running the Clerk with adjusted document types).
🚫
Do not approve blindly. The Judge Agent will only decide issues that appear in the matrix. A missing issue means no ruling on it. A biased framing means a biased analysis. Take time with this step — it is the most important human checkpoint in the entire pipeline.

Step 5 — Judge Agent (Judge Tab)

review_approved → judge_running → judge_done

After approval, the ▶ Run Judge Agent button appears. The Judge processes each issue sequentially — one LLM call per issue — then makes a final synthesis call to write the order.

Switch to the Judge tab to read the output once complete:

  • Background Facts — a narrative synthesised from the undisputed background.
  • Decisions on Issues — each issue has a collapsible card with Rule, Analysis, and Conclusion.
  • Final Order — the ultimate disposition, highlighted in a bordered box.

Step 6 — Export

Click the download icon in the top-right of the Judge tab to export the draft court order as a .md (Markdown) file. The exported file contains:

  • Case title as heading
  • Background facts section
  • Per-issue IRAC analysis with Rule, Analysis, and Conclusion
  • Final Order section

The Markdown file can be pasted into Word, Google Docs, or any legal drafting tool for final editing and formatting.

✅
Remember: The output is a draft. It is intended as a well-reasoned starting point for a human drafter or reviewing judge. Always apply independent legal judgment before finalising any order.

Troubleshooting

ProblemLikely causeFix
Clerk tab shows "No output yet" after running PageIndex was not run on the PDF, so no text was extracted Go to Documents, select the PDF, click ▶ and wait for processing to complete. Then re-run the Clerk.
Clerk facts look wrong or generic Document text was image-based or poorly OCR'd Check the Text tab for that document. If blank, the PDF is image-only and cannot be processed without OCR preprocessing.
Case status stuck at clerk_running API key missing or LLM request timed out Check the Activity Log in the sidebar for error messages. Verify OPENAI_API_KEY is set in .env.
Registrar matrix missing a key issue The issue was not raised in either clerk output Reject the matrix. Review the Clerk tab — if the issue is missing from clerk output, the source document may not mention it explicitly. Re-attach a more complete document and re-run Clerk.
Judge tab shows "Judge not run yet" after approval Server was not restarted after code update Restart the Flask server: python3 server.py
