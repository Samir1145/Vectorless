Registrar Agent

The Registrar reads both parties' structured submissions and produces the Adversarial Matrix — a neutral, issue-by-issue map of the entire dispute. This is the foundation on which the Judge Agent reasons.

Role and Responsibility

The Registrar does not analyse or decide — it organises. Its task is to answer one question: "What exactly are the parties arguing about, and on which specific points do they disagree?"

It reads both Clerk outputs simultaneously, identifies where the parties' positions diverge, and frames every contested point as a neutral legal issue with both stances mapped against it.

⚖️
The Adversarial Matrix is the most important document in the pipeline. A judge cannot decide a question that has not been properly framed. Everything the Judge says traces back to this matrix. This is why there is a mandatory human approval gate before the Judge runs.

Reading the Registrar Tab

Undisputed Background

This section lists the agreed-upon facts that set the scene. Check:

  • Every listed fact should genuinely be uncontested — if the Respondent disputes a fact, it should not be here.
  • Important background context (who the parties are, what the dispute concerns, key dates) should be present.
  • No legal conclusions should appear here — only facts.

Framed Issues Accordion

Each framed issue is a collapsible card. Click any issue to expand it. Inside, you will see two columns:

Petitioner Stance
The arguments the Petitioner makes on this specific issue, with the citations they invoke. This should reflect what the Petitioner actually argues — not a restatement of the issue.
Respondent Stance
The Respondent's counter-arguments on the same issue. If the Respondent has not addressed an issue (e.g. it was raised only by the Petitioner), this may legitimately be empty.

Review Criteria — What to Check

1
Completeness — are all issues captured?
Cross-reference the Issues Raised sections from the Clerk tab. Every issue raised by either party should appear as a framed issue in the matrix. A missing issue means no ruling on it.
2
Neutrality — is the framing balanced?
The neutral_issue_statement should not presuppose an answer. Avoid framings like "Whether the Respondent illegally refused…" — this assumes illegality. Prefer: "Whether the Respondent's refusal was lawful under…"
3
Accuracy — do stances reflect the actual submissions?
Compare the arguments listed under each stance against the corresponding Clerk output. The Registrar should not invent arguments that neither party made.
4
Citations are mapped to the right issue
A statute cited in a fact should not appear under an issue where it is irrelevant. Check that each citation listed under a stance is actually relied upon for that issue.
5
Undisputed facts are genuinely undisputed
If either party contests a fact in the background, it should be removed from undisputed_background and may need to become part of a framed issue's analysis.

Approving or Rejecting the Matrix

🚫
This is the only human control point in the pipeline. Once you click Approve, the Judge Agent will run autonomously. There is no mid-Judge pause. Make sure the matrix is correct before approving.

When to Approve

  • All issues are captured and correctly framed.
  • Each stance accurately reflects that party's actual arguments.
  • Undisputed facts are genuinely uncontested.
  • You are comfortable that the Judge can reason to a fair outcome based on this matrix alone.

When to Reject

  • A material legal issue is missing entirely.
  • Issue framing is biased toward one party.
  • A party's arguments are mischaracterised or missing.
  • A key statute or precedent is absent from the relevant issue's citations.
  • The undisputed background contains contested or inaccurate facts.

After Rejection

After rejecting, the case status becomes review_rejected. To proceed:

  1. Identify what was wrong — was it a Clerk extraction problem or a Registrar framing problem?
  2. If it was a Clerk problem, fix the source (re-attach a better document or re-run Clerk after PageIndex).
  3. Go back to the Clerk tab and click ▶ Run Registrar Agent again to produce a fresh matrix.
  4. Review again and approve or reject as appropriate.

Next Step

Once you click ✓ Approve, the ▶ Run Judge Agent button appears immediately below. Click it to start the final stage of the pipeline.
