Status Reference

A complete reference for every status value in the pipeline — what each one means, what triggered it, and what action (if any) is required from you.

Case Statuses

Each case moves through a defined set of statuses as it progresses through the pipeline. The current status is shown as a colour-coded dot next to the case in the left panel.

pending → run Clerk → clerk_running → Clerk done → clerk_done → run Registrar → registrar_running → Registrar done → review_pending
↓ Approve review_approved → run Judge → judge_running → Judge done → judge_done ✓
↓ Reject review_rejected → re-run Registrar
Any stage failure → error
StatusDot colourMeaningWhat to do
pending ● grey Case created, no agents have run. Documents may not yet be attached. Go to Setup tab, attach Petitioner and Respondent documents, click ▶ Run Clerk Agent.
clerk_running ● amber (pulse) Clerk Agent is currently processing both documents in parallel. LLM calls in progress. Wait. The Clerk tab will update automatically when done. Do not navigate away from the case.
clerk_done ● lime Both Clerk outputs are complete and saved. Registrar has not run yet. Review the Clerk tab output. If satisfactory, click ▶ Run Registrar Agent.
registrar_running ● amber (pulse) Registrar Agent is building the Adversarial Matrix. Single LLM call in progress. Wait. Registrar tab will populate when done.
review_pending ● blue The Adversarial Matrix is complete and awaiting human review. Action required. Open the Registrar tab, review the matrix carefully, then click ✓ Approve or ✗ Reject.
review_approved ● lime Human has approved the matrix. Judge Agent may now run. Click ▶ Run Judge Agent (visible in the Registrar tab).
review_rejected ● red Human has rejected the matrix. Pipeline is blocked until Registrar is re-run. Investigate why the matrix was wrong (Clerk issue or Registrar framing). Re-run Registrar Agent from the Clerk tab.
judge_running ● amber (pulse) Judge Agent is deliberating. Multiple sequential LLM calls in progress (one per issue + final order). Wait. This is the longest stage. The Judge tab will populate when done. Processing time scales with number of issues.
judge_done ● green Pipeline complete. The Draft Court Order is available in the Judge tab. Review the Judge tab. Export the order as Markdown using the download button.
error ● red An agent failed mid-run. The error message is shown in the Setup tab. Read the error message in the Setup tab. Check the Activity Log for details. Fix the underlying cause and re-run the failed stage.

Document Statuses

Individual documents attached to a case also have their own clerk processing status:

StatusMeaning
pendingDocument attached to case but Clerk has not run on it yet.
runningClerk is currently processing this specific document.
doneClerk extraction complete for this document.
errorClerk failed on this document. Check the Activity Log.

Transition Rules

Not every status transition is allowed. The following rules are enforced by the API:

ActionAllowed from statusBlocked if
Run Clerk AgentAny status (re-run allowed)Either document slot is empty in Setup
Run Registrar Agentclerk_done onlyClerk has not completed on both documents

Run Judge Agentreview_approved onlyAny other status — matrix must be explicitly approved

Sidebar Status Indicators

In the left panel Cases section, each case shows a small coloured dot:

Grey — pending / not started
Amber (pulsing) — agent running
Blue — review_pending (needs your action)
Lime — clerk_done / review_approved
Green — judge_done (complete)
Red — error / review_rejected
